Rule Runners - Claude Collaboration Summary

This page records the later design-review conversation around the browser game Rule Runners: what Codex tested live, what Claude changed, which ideas were rejected, and why the roster ultimately stopped at a stronger set of levels instead of being padded with weak ones.

Generated: March 17, 2026 Participants: User, Codex, Claude Mode: Live browser testing + level design review Outcome: 25 stronger levels, 1 app bug fixed

At A Glance

Bug Fix

Fixed the artifact error Identifier 'TS' has already been declared.

Accepted Level

The Course was added and signed off.

It teaches one rule handling two local conditions in sequence: gap detection, then ladder climbing.

Rejected Ideas

The Puddle and The Approach were both rejected.

Both sounded plausible at first, but collapsed under closer scrutiny into weaker or contradictory lessons.

Design Conclusion

The roster ended in a healthier place by stopping early.

Claude ultimately concluded that the remaining design space was too thin, too fragile, or required new mechanics.

Timeline

Phase 1 - Earlier polish and new levels
  • Codex reviewed earlier level fixes and rejected simplifications that removed the puzzle.
  • Two Gaps was kept as a jump-size tutorial, but its wording was made more honest.
  • The Ready was added as a state-change level built around parachutes and a fatal drop.
Phase 2 - App bug surfaced
  • The artifact pane displayed Uncaught SyntaxError: Identifier 'TS' has already been declared.
  • Codex explicitly asked Claude to treat it as a real bug, not preview noise.
  • Claude traced the problem to two concatenated HTML documents and removed the duplicate copy.
Phase 3 - The Course
  • Claude proposed a new mixed-local-conditions level.
  • The literal "fork" idea was rejected in favor of a cleaner sequential layout.
  • The first build failed: runners climbed up, faced left, and jumped back into the hazard.
  • Codex pushed for a geometry fix rather than a wording workaround.
  • Claude added a one-tile backstop at the ladder top, and the natural rule finally worked.
Phase 4 - Honest rejection of weaker ideas
  • The Puddle was rejected because hazard detection never became the clean core lesson.
  • The Approach was rejected because stopping near the exit contradicted the purpose of the exit.
  • Claude concluded that the remaining ideas were mostly thin sensor demos, fragile coordination puzzles, or features that needed new engine support.

Illustrations

Accepted Changes

Natural rule that signed off The Course:

if there is a gap ahead, jump. if there is a ladder ahead or you are on a ladder, climb up. otherwise move right

  • The console error was treated as a shipping bug, not an ignorable preview quirk.
  • The Course stayed in the game only after its geometry matched the intended lesson.
  • The game ended stronger because several tempting but weak concepts were rejected instead of being forced in.

Rejected Changes

  • The Puddle: hazard detection never became the clean core lesson; it either looped, collapsed into jumping, or was bypassed by simpler movement.
  • The Approach: proximity-to-exit stopping contradicted the basic purpose of the exit, so the lesson could not be made coherent.
  • Further single-sensor levels were judged too thin, while richer multi-creature ideas would need new sensors or more engine support.

Final Design Judgment

The most important outcome of the conversation was not just that one bug was fixed and one new level was added. It was that the review process stayed disciplined enough to reject filler. Claude ultimately concluded, and Codex agreed, that the game had reached a good stopping point: the remaining ideas were either thin novelties, fragile compiler puzzles, or concepts that needed new mechanics.

That left the game with a cleaner artifact, a better level roster, and a clearer design boundary around what counts as a genuinely worthwhile addition.
